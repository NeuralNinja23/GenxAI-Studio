# GenxAI Studio V4 — Clean-Room Refactor Master Blueprint

This blueprint governs the complete clean-room replacement of GenxAI Studio V3 with a **deterministic runtime with bounded cognition**.

V4 removes rigid cognitive phase dependency while preserving deterministic realization and validation pipelines. In its place, it establishes an abstract cognitive branch explorer, a canonical structural topology graph, dynamic attention routing, strict epistemic grounding, and transactional materialization.

---

## The Philosophy of Constrained Emergent Architecture

The fundamental architectural principle of GenxAI Studio V4 is **Constrained Emergent Architecture**. 

Instead of trying to "Generate an App" from freeform text, the V4 system operates to **Evolve a Valid Topology**. Under this model:
* **The frontend is topology.**
* **The backend is topology.**
* **API routes, database models, workflows, and layout structures are topology.**
* **Files are merely projections of the topology graph onto the disk.**

Emergence is permitted only within a bounded, governed topology mutation space.

### Bounded Emergence Boundaries:
* **What the System IS Permitted to Evolve:** Structure, workflow routes, UI panel layouts, database logic mappings, and isolated logic components.
* **What the System IS NOT Permitted to Evolve:** The core execution kernel, transaction engines, active oracle validation hierarchies, stable framework substrates, and security bounds.

## Core Laws of V4 Runtime Governance

---

### 1. No Textual Authority Principle

Natural language outputs possess zero execution authority.

Text generated by cognitive faculties may:
* propose mutations,
* suggest semantic refinements,
* annotate reasoning traces.

Text may NEVER:
* directly generate filesystem structures,
* define canonical topology,
* bypass AST validation,
* override oracle outcomes,
* initiate transaction commits.

---

### 2. Reality Priority Stack

To prevent semantic hallucinations from overriding actual runtime truth, the V4 runtime operates under a strict **Reality Priority Stack**:

1. **Runtime execution truth** (Authoritative raw execution output and logs)
2. **Oracle hard validation** (Syntactic, topological, and compilation passes)
3. **Topology integrity** (Canonical project graph structure)
4. **AST validity** (Syntactically correct abstract syntax trees)
5. **Projection synchronization** (Conformity between AST and disk state)
6. **Semantic alignment** (Adherence to intent guidelines)
7. **Cognitive scoring** (Attention scores and convergence slope)
8. **LLM reasoning** (Lowest priority; purely abstract proposals)

*Higher layers always override lower layers.*

---

### 3. Canonical Reality Sources

V4 establishes a strict ontological model of software reality:

* **Semantic Truth:** `IntentField`
* **Structural Truth:** `ProjectTopologyGraph`
* **Syntactic Truth:** `AST Graph`
* **Persistence Truth:** `Filesystem Projection`
* **Execution Truth:** `Runtime State + Oracle Evidence`

*No lower layer may redefine a higher layer.*

---

### 4. Projection Ownership Hierarchy

V4 enforces strict separation of layers to block invalid circular dependencies or state corruption:

* **IntentField** owns semantics.
* **ProjectTopologyGraph** owns structure.
* **AST** owns realizable syntax.
* **Filesystem** owns persisted artifacts.
* **Runtime** owns execution state.

*Lower layers may NEVER mutate higher layers directly.*
* Filesystem changes cannot alter AST truth.
* AST changes cannot alter topology truth.
* Topology changes cannot alter semantic intent boundaries.

---

### 5. Deterministic Projection Boundary

All filesystem writes MUST originate exclusively from:

$$\text{ProjectTopologyGraph} \longrightarrow \text{AST Generator} \longrightarrow \text{AST Mutator} \longrightarrow \text{AST Validator} \longrightarrow \text{AST Projector}$$

Cognitive faculties are permanently forbidden from:
* writing files directly
* mutating runtime state
* persisting AST structures
* executing shell commands
* performing transaction commits

---

### 6. Runtime Determinism Boundary

Cognitive exploration may be non-deterministic.

Runtime realization MUST be deterministic.

Identical:
* topology,
* AST,
* contracts,
* mutations

must always produce identical filesystem projections on the disk.

---

### 7. Immutable Runtime Kernel

To block recursive self-corruption and ensure absolute structural safety, the runtime governance infrastructure is **immutable** during execution. Cognitive mutation systems are permanently prohibited from modifying:
* `execution_kernel.py`
* `transaction_engine.py`
* `topology_validator.py`
* `runtime_projection_validator.py`
* Oracle hierarchy tiers
* Lease systems
* Sandbox isolation parameters
* Rollback engine configurations

## Cognitive Mutation Space

---

### 1. Mutation Classes

To structure the evolution of structural schemas, mutations are classified into strict tiers:

* **Tier 1 — Cosmetic Mutation:** Spacing, typography, design element alignment, and layout density.
* **Tier 2 — Structural UI Mutation:** Component hierarchies, layout panel segmentations, and interaction routing.
* **Tier 3 — Behavioral Mutation:** Local workflow logic, state propagation paths, and API interaction flows.
* **Tier 4 — Topology Mutation:** Route introductions, database schema relationships, and service decompositions.
* **Tier 5 — Forbidden Mutation:** Changes targeting the runtime kernel, oracle hierarchy, transaction systems, or sandbox boundaries (Blocked by the `Constraint Engine`).

---

### 2. Mutation Cost Functions

Branch exploration is governed by a weighted optimization score:

$$\text{Cost} = w_1(\text{Topology Drift}) + w_2(\text{Oracle Failure Weight}) + w_3(\text{Runtime Instability}) + w_4(\text{Structural Complexity}) + w_5(\text{Failure Repulsion Similarity}) + w_6(\text{Projection Entropy})$$

*Lower cost branches receive higher exploration priority.*

---

### 3. Branch Collapse Conditions

A branch state is terminated immediately and undergoes **Branch Invalidation** if:
* Any hard oracle validation failure persists.
* The topology graph becomes cyclic.
* Projection divergence between AST and disk exceeds safety thresholds.
* Mutation entropy stagnates ($\Delta L == 0$ over consecutive cycles).
* Runtime synchronization checks fail.
* Similarity to SQLite failure memory vector coordinates exceeds repulsion safety thresholds.

## Core Execution Invariant: "Cognition Proposes — Runtime Decides"

V4 establishes a total separation of concerns between cognitive exploration and runtime execution. **Cognition is non-authoritative.**

```
       [ COGNITIVE LAYER ]                     [ RUNTIME LAYER ]
       (Non-Authoritative)                       (Authoritative)
               │                                        │
     Proposes Bounded Mutation                          │
               │                                        ▼
               └───────────────────────────────► [ Constraint Engine ]
                                                        │
                                                 Validation Passed
                                                        │
                                                        ▼
                                                 [ AST Engine ]
                                                        │
                                                        ▼
                                              [ AST Projector ]
                                                        │
                                                        ▼
                                            [ Projection Validator ]
                                                        │
                                                        ▼
                                               [ Oracle Pipeline ]
                                                        │
                                                All Oracles Passed
                                                        │
                                                        ▼
                                            [ Transaction Commit ]
```

### Cognitive Layer (Proposes Only)
* **Authorized to:** explore branch states, score paths, propose mutations, plan alternative topologies, and analyze convergence slopes.
* **PROHIBITED from:** writing to the filesystem, executing shell commands, performing database state updates, running container tasks, or validating code.

### Runtime Layer (Authoritative)
* **Authorized to:** materialise AST structures, validate structural constraints, manage execution leases, trigger rollback routines, run test frameworks, and commit filesystem transactions.
* **Only the Runtime Kernel and the Oracle Hierarchy possess execution authority.**

```
   [ Stage 1: Freeze Runtime ]

                │
                ▼ Passed Verification tests
   [ Stage 2: Canonical Topology ]

                │
                ▼ Passed Verification tests
   [ Stage 3: AST Pipeline ]
   (AST generators, mutators, mergers, AST projectors, and validators)
                │
                ▼ Passed Verification tests
   [ Stage 4: Oracle Layer ]
   (Syntax, topology, and runtime oracles, projection validator)
                │
                ▼ Passed Verification tests
   [ Stage 5: Runtime Synchronization ]

                │
```
